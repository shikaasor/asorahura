**src/app/engage/actions.ts — rewrite with score routing:**
- Keep `"use server"` directive
- Add `import { redirect } from "next/navigation"`
- Rename function to `submitInquiry` (keep same name — page.tsx imports this)
- Read score from FormData: `const scoreRaw = formData.get("score") as string; const score = parseInt(scoreRaw || "0", 10);`
- Read all other fields same as before (name, email, company, role, companySize, operationalVolume, challenge, timeline, budget, context)
- Keep the existing Google Script URL submission logic (preserve CRM logging)
- After the fetch (fire-and-forget the CRM logging, don't block routing on it):
  - If `score >= 70`: `redirect("/checkout?tier=strategy")`
  - Else if `score >= 40`: `redirect("/checkout?tier=discovery")`
  - Else: `redirect("/engage/confirmation")`
- Important: `redirect()` in Next.js throws internally — do NOT wrap it in try/catch that catches redirect throws. Wrap only the fetch in try/catch, then call redirect after.
- If score is 0 or NaN (no score param passed): redirect to /engage/confirmation as safe fallback.
- Return type changes: the function no longer returns a value in the redirect paths. Add return type `Promise<never | { success: boolean; message: string }>` — or simply use `Promise<void>` and let redirect handle all paths.
