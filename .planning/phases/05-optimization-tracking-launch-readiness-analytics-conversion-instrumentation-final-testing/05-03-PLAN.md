  <files>lighthouserc.js</files>
  <action>
    Install Lighthouse CI globally:
    npm install -g @lhci/cli@latest

    Create lighthouserc.js at the repository root:

    ```javascript
    // lighthouserc.js
    // Run against Vercel preview URL: VERCEL_URL=your-preview.vercel.app lhci autorun
    module.exports = {
      ci: {
        collect: {
          url: [
            process.env.VERCEL_URL
              ? `https://${process.env.VERCEL_URL}`
              : 'http://localhost:3000',
            process.env.VERCEL_URL
              ? `https://${process.env.VERCEL_URL}/assessment/deep`
              : 'http://localhost:3000/assessment/deep',
            process.env.VERCEL_URL
              ? `https://${process.env.VERCEL_URL}/checkout`
              : 'http://localhost:3000/checkout',
          ],
          numberOfRuns: 3,
        },
        upload: {
          target: 'temporary-public-storage',
        },
        assert: {
          assertions: {
            // Soft targets — warn only (launch not blocked if missed)
            'categories:performance': ['warn', { minScore: 0.70 }],
            'categories:accessibility': ['warn', { minScore: 0.85 }],
            'categories:best-practices': ['warn', { minScore: 0.80 }],
            'categories:seo': ['warn', { minScore: 0.85 }],
          },
        },
      },
    };
    ```

    Then deploy current code to Vercel preview:
    Run: vercel --yes
    This generates a preview URL (e.g., asorahura-git-main-xxx.vercel.app).

    Then run Lighthouse against the preview URL:
    VERCEL_URL=asorahura-git-main-xxx.vercel.app lhci autorun

    If lhci is not available as a command: npx lhci autorun

    Record the output scores for homepage, assessment/deep, and checkout in the test results file (Task 3). Target thresholds (soft — non-blocking):
    - Homepage: Performance >70, LCP <2s, CLS <0.1
    - Assessment results (/assessment/deep): Performance >70, LCP <1.5s
    - Checkout: Performance >70, LCP <2s

    If Vercel CLI is not authenticated, Claude will create a checkpoint for authentication. Do not skip this step — localhost Lighthouse scores are meaningless per user decision (real CDN conditions required).
